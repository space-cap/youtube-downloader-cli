### Day 2: 다운로드 API 구현
**소요 시간**: 약 4시간  
**이슈**: #2

#### 구현 내용
- 작업 관리 시스템 (`tasks.py`)
  - Task ID 생성 (UUID)
  - 작업 상태 관리 (pending → downloading → completed/failed)
  - 진행률 업데이트 기능
- 다운로드 API 엔드포인트
  - `POST /api/v1/download`: 다운로드 시작
  - `GET /api/v1/download/{task_id}/status`: 상태 조회
  - `GET /api/v1/download/{task_id}/file`: 파일 다운로드
- 백그라운드 다운로드 작업
  - 기존 `Downloader` 클래스 통합
